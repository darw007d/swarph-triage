# swarph-triage 0.1.0 — implementation plan

**Goal:** fill the 12 `NotImplementedError` stubs + add real test coverage → `0.1.0` stable (functional, pip-installable). The contract is FROZEN by the README + the already-implemented `schema.py` / `config.py` / `state_machine.py`; do not change it.

**Reference source (proven logic to port):** the production error-triage this was extracted from lives at `/root/hedge-fund-mcp/workers/triage_queue.py` (fingerprint UPSERT + count/last_seen maintenance + regression-resurrect + priority-by-severity×freq×recency). Port the *patterns*, not the journalctl/SQLite specifics — swarph-triage is backend-agnostic via **SQLAlchemy Core** (`schema.py` tables: `swarph_triage_fingerprints`, `swarph_triage_occurrences`, `swarph_triage_state_log`).

**Formula (from `priority.py` docstring + `config.py`):**
`priority = severity_weights[severity] * freq_curve(count_24h) * decay(hours_since_last_seen) * max(actionability, actionability_floor)`, clamped to `[priority_min, priority_max]`. `freq_curve="log"` → `log(1+count_24h, freq_log_base)`. `decay` → `exp(-hours_since_last_seen * ln2 / decay_half_life_hours)`. If `cooldown_until` is in the future, ramp the score toward 0 (linear from cooldown-start to cooldown-end). All coefficients from the merged `config` — no hardcoded constants.

**Tech:** Python 3.10+, SQLAlchemy Core (already a dep), stdlib `argparse`/`math`/`datetime`; `pytest`. FastAPI is an optional extra (`[fastapi]`).

**Test strategy:** real round-trips against `sqlite:///:memory:` (or a tmp file) via `swarph_triage.open(...)`. Inject `now_ts`/`now` everywhere time matters (the stubs already take `now_ts`) so decay/cooldown/regression-grace are deterministic. Mirror the existing `tests/test_smoke.py` import/style.
